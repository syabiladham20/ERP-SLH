Recommendations for 2026 Expansion
Database Migration Required

The concurrency test revealed significant bottlenecks due to SQLite's single-writer locking mechanism. With 10 concurrent users, response times averaged 10.8 seconds. Recommendation: Migrate to PostgreSQL or MySQL to enable row-level locking and concurrent writes.

Archiving Strategy

Archiving logic has been implemented. Inactive flocks are now excluded from main dashboard queries, reducing the load as historical data grows. A dedicated History View allows access to closed flocks.

Application Logic

SQL-Based Aggregation has been implemented for the Executive Dashboard ISO Report, moving processing from Python loops to the Database Engine. This reduced load times by ~30% total. However, the system is now strictly bound by Database IOPS and SQLite concurrency limits.
